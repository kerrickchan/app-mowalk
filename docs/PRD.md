## 1. 项目概述

### 1.1 产品背景

Android 智能手机已普遍内置高精度计步传感器（`TYPE_STEP_COUNTER`）。本产品**仅针对手机设备**，不连接任何手表、手环或外部心率设备，利用手机自身的传感器能力，为用户提供自动、低耗电的步数记录与健康数据管理服务，并通过 Health Connect 与其他健康应用安全互通。

### 1.2 产品目标

- 提供可靠的**后台计步**功能，完全基于手机硬件，不依赖任何外部穿戴设备。
- 记录每日步数、行走距离、卡路里消耗，并支持历史趋势查看。
- 通过 **Health Connect** 实现步数数据的读写与同步（可选），同时保留本地 Room 存储。
- 严格遵循 Android 隐私与权限最佳实践，所有健康数据留在手机本地，不上传任何云端。

### 1.3 范围限定

- ✅ **包含**：手机内置计步传感器（`SensorManager`）、前台服务、本地数据库、Health Connect 集成、图表统计、数据导出。
- ❌ **不包含**：任何与手表、手环、心率带等外部设备的连接；BLE 扫描/配对；心率、血氧、血压等生理指标采集。

### 1.4 名词定义

- **Health Connect**：Android 官方的健康数据枢纽，安装在手机上的系统级应用。
- **SensorManager**：Android 系统 API，用于访问手机硬件传感器。
- **前台服务**：Android 组件，保证应用在后台或熄屏时仍能持续计步。

## 3. 功能需求

### 3.1 计步功能（手机传感器）

| ID    | 需求描述                                                                                                   | 优先级 |
| :---- | :--------------------------------------------------------------------------------------------------------- | :----- |
| F‑01 | 通过 `SensorManager.TYPE_STEP_COUNTER` 获取手机开机以来的总步数，并计算每次活动的增量步数。              | P0     |
| F‑02 | 提供**前台服务**，应用退到后台或屏幕关闭后持续计步。服务通知栏提示“计步中”，用户可点击关闭服务。   | P0     |
| F‑03 | 展示**今日步数**、本周步数趋势图、本月日历视图。                                                     | P0     |
| F‑04 | 允许用户输入身高（cm）和体重（kg），自动估算行走距离（米）和消耗卡路里（千卡）。                           | P1     |
| F‑05 | 支持手动修正某一天的历史步数（用于传感器误差或用户主动调整）。                                             | P2     |
| F‑06 | 若手机不支持 `TYPE_STEP_COUNTER`，降级为基于 `TYPE_ACCELEROMETER` 的简单步伐检测（需说明准确性下降）。 | P2     |

### 3.2 数据存储与健康互联

| ID    | 需求描述                                                                                                          | 优先级 |
| :---- | :---------------------------------------------------------------------------------------------------------------- | :----- |
| F‑07 | 使用**Room** 数据库本地持久化存储步数记录（每天一条记录）及用户身体指标。                                   | P0     |
| F‑08 | 集成**Health Connect SDK**，提供双向同步：读取其他应用写入的步数，也将本应用记录的步数写入 Health Connect。 | P0     |
| F‑09 | 在设置页提供开关：“同步至 Health Connect”，默认开启；关闭后本应用只读写本地数据库。                             | P1     |
| F‑10 | 支持将全部步数历史导出为 CSV 文件，保存到手机 Download 目录。                                                     | P2     |

### 3.3 权限与隐私管理

| ID    | 需求描述                                                                                        | 优先级 |
| :---- | :---------------------------------------------------------------------------------------------- | :----- |
| F‑11 | 首次启动时请求 `ACTIVITY_RECOGNITION` 权限（计步必需），附有清晰的说明文字。                  | P0     |
| F‑12 | 若用户需要后台读取 Health Connect，在相关功能入口请求 `READ_HEALTH_DATA_IN_BACKGROUND` 权限。 | P1     |
| F‑13 | 设置页提供“清除所有本地数据”按钮，方便用户彻底删除健康信息。                                  | P1     |

### 3.4 UI/UX 要求（手机专属）

| ID    | 需求描述                                                                         | 优先级 |
| :---- | :------------------------------------------------------------------------------- | :----- |
| F‑14 | 主仪表板显示：今日步数（大数字）、距离（km）、卡路里（kcal），以及一个刷新按钮。 | P0     |
| F‑15 | 步数趋势使用折线图（本周/本月切换），支持点击某天查看详细步数。                  | P1     |

## 4. 非功能性需求

### 4.1 性能

- 后台计步额外耗电 ≤ 2% / 小时（主流设备，熄屏状态）。
- 传感器回调延迟 ≤ 500ms，UI 步数刷新 ≤ 1 秒。
- 应用冷启动时间 ≤ 2 秒。

### 4.2 可靠性

- 前台服务崩溃后自动重启（`START_STICKY`）。
- Health Connect 读写失败时自动重试 3 次（指数退避）。
- 数据库升级必须保证历史数据不丢失，提供迁移脚本。

### 4.3 隐私与安全

- 所有步数、身体数据**绝不主动上传**至任何互联网服务器，仅存储在手机本地或 Health Connect（仍留在设备端）。
- 日志中禁止输出原始步数或任何可识别用户身份的信息。

### 4.4 兼容性

- 最低支持 Android 8.0 (API 26)，目标 API 34+。
- 仅支持手机（Phone 形态），不针对平板或可折叠屏做强制适配，但应保证基本可用。

## 5. 技术约束与建议

| 模块         | 推荐技术 / API                                  | 说明                                                   |
| :----------- | :---------------------------------------------- | :----------------------------------------------------- |
| 计步传感器   | `SensorManager` + `TYPE_STEP_COUNTER`       | 直接获取系统步数，最可靠                               |
| 前台服务     | `ForegroundService` + `Service`             | Android 14+ 需 `FOREGROUND_SERVICE_HEALTH` 权限      |
| 本地存储     | Room + Kotlin Coroutines Flow                   | 单表 `daily_steps` (date, steps, distance, calories) |
| 健康数据互通 | Health Connect SDK (1.2+)                       | 需手机已安装 Health Connect 应用                       |
| 图表库       | MPAndroidChart 或 Compose 自定义绘制            | 推荐 `androidx.compose.material3` + 轻量图表         |
| 权限管理     | Google Accompanist 或官方 `PermissionRequest` | 统一处理运行时权限                                     |
| 后台任务     | WorkManager（每隔 6 小时同步 Health Connect）   | 用于从 Health Connect 拉取其他应用写入的步数           |

## 7. 风险与依赖

| 风险                                 | 影响                     | 缓解措施                                         |
| :----------------------------------- | :----------------------- | :----------------------------------------------- |
| 部分老旧手机无 `TYPE_STEP_COUNTER` | 计步不可用               | 提示用户，并降级为加速度计算法（准确性较低）     |
| 用户未安装 Health Connect            | 无法同步                 | 引导跳转 Google Play 下载，本机数据仍正常记录    |
| Android 系统杀后台                   | 前台服务被停止，步数漏记 | 引导用户加入“忽略电池优化”白名单，显示持续通知 |
| Android 16+ 后台限制更严             | 前台服务可能无法常驻     | 提供“手动同步”按钮作为备选，缩短同步间隔不适用 |
